# Requisitos não funcionais

## Performance

O Website deverá apresentar respostas rápidas às respostas do usuário, carregamento eficiente das questões e estatísticas garantindo boa experiência mesmo com conexões mais lentas

## Segurança

Autenticação e autorização garantindo que o usuário acesse apenas os dados que pertencem a ele.

## Usabilidade

O website deverá ser de fácil utilização, intuitiva, responsiva e acessível facilitando o uso para estudantes com diferentes níveis de familiaridade com a tecnologia.

## Compatibilidade

Deverá ser compatível com a maioria dos navegadores e sistemas operacionais.

## Dependencias

### Interna

Para o Front-End React com TypeScript, Back-End em C# com [ASP.NET](http://asp.net) Core, Entity Framework Core e um banco de dados para armazenar os dados do usuário.

### Externa

[API de questões do ENEM](https://docs.enem.dev/introduction) para obtenção de questões, provas, alternativas e metadados.
